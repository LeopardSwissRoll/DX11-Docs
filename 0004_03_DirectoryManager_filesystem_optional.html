DirectoryManager +
C++17 filesystem, optional

로그 파일을 어디에 저장할지? 경로를 관리하는 매니저. std::filesystem과 std::optional이 처음 등장한다.

DirectoryManager 전체 구조

C++ 클래스 선언 Core/DirectoryManager.h
class DirectoryManager : public Singleton<DirectoryManager>
{
    friend Singleton<DirectoryManager>;

private:
    std::filesystem::path _rootPath;        // 실행 파일이 있는 경로
    std::filesystem::path _rootParentPath;  // 루트의 부모 경로

    // 자주 사용하는 경로들은 미리 캐싱해놓는다
    std::unordered_map<std::string, std::filesystem::path> _paths;

private:
    void RegisterPath(const std::string& path);

public:
    void Init();
    virtual void Destroy() override;

    const std::filesystem::path& GetRootPath() const { return _rootPath; }
    std::optional<std::filesystem::path> GetPath(const std::string& pathName) const;
};

Init — 루트 경로 설정

C++ Init() Core/DirectoryManager.cpp
void DirectoryManager::Init()
{
    // 실행 파일이 있는 경로
    _rootPath = std::filesystem::current_path();
    if (!std::filesystem::exists(_rootPath))       //사실 굳이 필요 없긴 하다
        return;

    if (!std::filesystem::is_directory(_rootPath)) //사실 굳이 필요 없긴 하다.
        return;

    // 부모경로 --> 이전 경로
    _rootParentPath = _rootPath.parent_path();
    if (!std::filesystem::exists(_rootParentPath))      //사실 굳이 필요 없긴 하다.
        return;

    if (!std::filesystem::is_directory(_rootParentPath)) //사실 굳이 필요 없긴 하다.
        return;

    RegisterPath("Logs");
}
강사님 주석: "사실 굳이 필요 없긴 하다"
current_path()는 반드시 존재하는 디렉토리를 반환하니까 exists/is_directory 체크가 불필요해. 방어적 프로그래밍 차원에서 넣은 거.
Init 시 경로 구조
C:\Users\Game\Build\Bin
← _rootPath (current_path)
↑ parent_path()
C:\Users\Game\Build
← _rootParentPath
↓ / "Logs"
C:\Users\Game\Build\Logs
← RegisterPath("Logs")

std::filesystem::path — 경로 전용 타입

std::string으로 경로를 다루면 OS마다 구분자가 달라서 (\ vs /) 문제가 생겨. std::filesystem::path는 이걸 자동으로 처리해줘.

핵심: / 연산자로 경로 결합

C++ 경로 결합
std::filesystem::path registeredPath = _rootParentPath / path;

// _rootParentPath = "C:\Users\Game\Build"
// path = "Logs"
// 결과 = "C:\Users\Game\Build\Logs"   ← 자동으로 \ 추가!
std::string으로 하면
root + "\\" + path
구분자 직접 관리. Linux에서는 "/"로 바꿔야...
filesystem::path
root / path
OS에 맞는 구분자 자동 처리!

자주 쓰는 filesystem 함수들

함수하는 일
current_path()현재 작업 디렉토리 (실행 파일 위치)
exists(path)경로가 존재하는지
is_directory(path)디렉토리인지 (파일이 아니라)
create_directories(path)중간 폴더까지 전부 생성 (mkdir -p)
path.parent_path()부모 디렉토리 경로

RegisterPath — 경로 등록 + 자동 생성

C++ RegisterPath Core/DirectoryManager.cpp
void DirectoryManager::RegisterPath(const std::string& path)
{
    std::filesystem::path registeredPath = _rootParentPath / path;
    if (!std::filesystem::exists(registeredPath))
        std::filesystem::create_directories(registeredPath);

    if (!std::filesystem::is_directory(registeredPath))
        return;

    _paths[path] = registeredPath;
}
1
_rootParentPath / path로 전체 경로 조합
2
폴더가 없으면 create_directories로 생성 (중간 폴더까지)
3
디렉토리가 맞으면 _paths 맵에 캐싱

std::optional — "있을 수도 있고, 없을 수도 있는 값"

왜 필요한가?

포인터로 표현
path* Get(name);
if (p) use(*p);

댕글링 위험, 소유권 불명확
bool + out 매개변수
bool Get(name, path& out);
if (Get(n, p)) use(p);

안전하지만 번거로움
std::optional (C++17)
optional<path> Get(name);
if (p) use(*p);

안전 + 깔끔!

GetPath 구현

C++ GetPath — optional 반환 Core/DirectoryManager.cpp
std::optional<std::filesystem::path> DirectoryManager::GetPath(const std::string& pathName) const
{
    auto it = _paths.find(pathName);
    if (it == _paths.end())
        return std::nullopt; // 없다.

    return it->second;
}

optional 사용법

C++ LogManager에서의 사용 패턴
// LogManager::OpenNewFile() 에서:
auto logPath = DirectoryManager::Instance().GetPath("Logs");
if (!logPath)      // optional은 bool로 변환 가능 (값 있으면 true)
    return;

// 값 꺼내기 — 두 가지 방법:
*logPath           // 역참조 연산자
logPath.value()    // .value() 메서드 (없으면 예외 발생)
optional의 두 가지 상태
값이 있을 때
C:\...\Build\Logs
if (logPath) → true
*logPath → 경로 반환
값이 없을 때 (nullopt)
std::nullopt
if (logPath) → false
*logPath → 미정의 동작!
