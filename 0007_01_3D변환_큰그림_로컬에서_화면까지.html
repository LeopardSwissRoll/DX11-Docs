Render() — 변환이 추가된 흐름

C++ 핵심 부분 Core/GameEngine.cpp
FMatrix mW, mP;
FMatrix mS, mR, mT;

mS.Scaling(scale);
mR.Rotation(rot);
mT.Translation(pos);

mW = mS * mR * mT;

//투영생성
mP = DirectX::XMMatrixPerspectiveFovLH(DirectX::XMConvertToRadians(90.f), 1280.f / 720.f, 0.5, 1000.f);

buffer->SetWorldMatrix(mW);
buffer->SetProjMatrix(mP);

buffer->Update();

shader->SetShader();
mesh->Render();

4

5

월드 행렬 = S × R × T

C++
mW = mS * mR * mT;  // 크기 → 회전 → 이동
S × R × T (정상)

크기 조절 → 제자리 회전 → 이동

T × R × S (비정상)

이동 → 원점 기준 공전 → 크기 (의도 아님!)

행렬 곱은 교환법칙이 안 됨. 순서를 바꾸면 완전히 다른 결과.

자전 vs 공전 — 직접 확인

슬라이더로 회전각을 조절해봐. 왼쪽은 자전(S×R×T), 오른쪽은 공전(T×R×S).

S × R × T (자전)
T × R × S (공전)

원근 투영 — PerspectiveFovLH

C++
mP = DirectX::XMMatrixPerspectiveFovLH(
    DirectX::XMConvertToRadians(90.f),  // FOV 90도
    1280.f / 720.f,                      // 종횡비 16:9
    0.5,                                 // near plane
    1000.f                               // far plane
);
인자 설명
LH = Left-Handed (z가 화면 안쪽 증가)
FOV 90° = 넓은 시야. 60°면 줌인 효과
near/far = 이 범위 밖은 클리핑(잘림). pos._z = 10.f로 삼각형을 범위 안에 배치

입력 처리 — GetAsyncKeyState

C++
if (GetAsyncKeyState('D') & 0x8000)
    pos._x += 1.f * deltaTime;
