핵심: GameEngine::Render()

이 함수가 매 프레임 실행되면서 화면에 삼각형이 그려져.

C++ 매 프레임 렌더링 Core/GameEngine.cpp
void GameEngine::Render(float deltaTime)
{
    Device::Instance().ClearBackBuffer(_clearColor);
    Device::Instance().ClearDepthStencil(1.f, 0);
    Device::Instance().SetTarget();

    Ptr<Shader> shader = ShaderManager::Instance().FindShader("ColorMeshShader");
    Ptr<Mesh> mesh = AssetManager::Instance().Get<MeshManager>(eAssetType::Mesh)->FindMesh
    ("Triangle");

    shader->SetShader();
    mesh->Render();

    Device::Instance().Render();
}

한 줄씩 추적 — 파이프라인 매핑

1. Clear — 이전 프레임 지우기

C++ Core/Device.cpp
void Device::ClearBackBuffer(float clearColor[4])
{
    //백버퍼 매개변수로 들어온 색상으로 만듭니다.
    _context->ClearRenderTargetView(_targetView.Get(), clearColor);
}

void Device::ClearDepthStencil(float depth, uint8 stencil)
{
    //뎁스/스탠실 뷰를 들어온 매개변수로 밀어줍니다.
    _context->ClearDepthStencilView(_depthStencil.Get(), D3D11_CLEAR_DEPTH | D3D11_CLEAR_STENCIL, depth, stencil);
}

이전 프레임 지우기. _clearColor = {0, 0, 0, 1} (검정).

2. SetTarget — OM 바인딩

C++ Core/Device.cpp
void Device::SetTarget()
{
    //랜더링 파이프라인 과정에서 마지막에 그림을 출력시킬 목적지를 지정
    _context->OMSetRenderTargets(1, _targetView.GetAddressOf(), _depthStencil.Get());
}

3. 셰이더 + 메시 가져오기

InitManager에서 미리 만들어둔 셰이더와 메시를 이름으로 검색. 매 프레임 생성이 아니라 캐싱된 걸 가져옴.

4. shader→SetShader() — VS + PS + InputLayout 바인딩

C++ Shader/GraphicShader.cpp
void GraphicShader::SetShader()
{
    Device::Instance().GetContext()->VSSetShader(_vs.Get(), nullptr, 0);
    Device::Instance().GetContext()->PSSetShader(_ps.Get(), nullptr, 0);

    Device::Instance().GetContext()->IASetInputLayout(_inputLayout.Get());
}

5. mesh→Render() — IA 바인딩 + Draw

C++ Core/Mesh.cpp
void Mesh::Render()
{
    uint32 stride = _vertexBuffer._size;
    uint32 offset = 0;

    Device::Instance().GetContext()->IASetPrimitiveTopology(_primitive);
    Device::Instance().GetContext()->IASetVertexBuffers(0, 1, _vertexBuffer._buffer.GetAddressOf(), &stride, &offset);

    Device::Instance().GetContext()->IASetIndexBuffer(_indexBuffer._buffer.Get(), _indexBuffer._fmt, 0);

    Device::Instance().GetContext()->DrawIndexed(_indexBuffer._count, 0, 0);
}

DrawIndexed(3, 0, 0) — 이 순간 GPU가 파이프라인 전체를 실행: IA→VS→RS→PS→OM.

6. Device::Render() — Present

C++ Core/Device.cpp
void Device::Render()
{
    _swapChain->Present(0, 0);
}

코드 ↔ 파이프라인 매핑

Render() 코드단계하는 일
ClearBackBuffer()사전백버퍼를 검정으로 초기화
ClearDepthStencil()사전깊이 버퍼를 1.0으로 리셋
OMSetRenderTargets()OMRTV + DSV를 출력 대상으로 지정
VSSetShader()VS버텍스 셰이더 바인딩
PSSetShader()PS픽셀 셰이더 바인딩
IASetInputLayout()IA정점 해석 방법 설정
IASetPrimitiveTopology()IA"삼각형 리스트"로 해석
IASetVertexBuffers()IA정점 데이터 바인딩
IASetIndexBuffer()IA인덱스 데이터 바인딩
DrawIndexed(3)전체GPU가 IA→VS→RS→PS→OM 실행!
Present()후처리백↔프론트 버퍼 교체

삼각형이 빨+초+파 인 이유

C++ 정점별 색상 Core/MeshManager.cpp
FVertexColor triangle[3]
{
    FVertexColor(0.f, 0.5f, 0.f, 1.f, 0.f, 0.f, 1.f),     // 위: 빨강 (1,0,0)
    FVertexColor(0.5f, -0.5f, 0.f, 0.f, 1.f, 0.f, 1.f),   // 오른쪽: 초록 (0,1,0)
    FVertexColor(-0.5f, -0.5f, 0.f, 0.f, 0.f, 1.f, 1.f)  // 왼쪽: 파랑 (0,0,1)
};

래스터라이저(RS)가 꼭짓점 사이의 픽셀을 보간(interpolation)하니까, 삼각형 내부가 그라데이션으로 칠해져.

렌더링 결과 — NDC 좌표계 위의 삼각형
● v0 (빨강) ● v1 (초록) ● v2 (파랑)

MemoryPool UB 수정

이전 (UB!)
C++
void* operator new(size_t size)
{
    FSegment seg = ....Rent((int)size);
    Object* obj = (Object*)seg._Data;
    obj->_Segment = seg; // ← 생성자 전!
    return seg._Data;
}
현재 (안전)
C++
void* operator new(size_t size)
{
    FSegment seg = ....Rent((int)size);
    return seg._Data; // 객체 안 건드림
}
void operator delete(void* ptr)
{
    ....Return(ptr); // _rentals에서 검색
}

MemoryPool이 _rentals 맵(unordered_map<void*, FSegment>)으로 내부에서 대여 기록 추적. operator new/delete가 _Segment 멤버를 더 이상 사용하지 않음.

왜 UB인지 — 메모리 관점
이전 — 생성자 전 접근
seg._Data 메모리
??? (미초기화)
↑ obj->_Segment = seg
생성자 전에 접근! UB!
현재 — 외부 추적
seg._Data 메모리
(건드리지 않음)
↓ _rentals[ptr] = seg
