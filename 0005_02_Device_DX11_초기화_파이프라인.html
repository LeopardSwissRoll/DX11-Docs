Device가 들고 있는 DX11 객체 5개

객체비유역할
ID3D11Device공장GPU 리소스(텍스처, 버퍼, 셰이더)를 생성
ID3D11DeviceContext작업자GPU에 렌더링 명령. 파이프라인에 리소스 바인딩
IDXGISwapChain무대 스크린더블 버퍼링 — 백/프론트 버퍼 교체
ID3D11RenderTargetView캔버스백버퍼를 렌더링 목적지로 지정
ID3D11DepthStencilView깊이 지도각 픽셀의 깊이 + 스텐실 저장

HRESULT 패턴

DX11 함수는 거의 다 HRESULT를 반환해. FAILED() 매크로로 체크하는 패턴이 Init 전체에 반복됨:

C++ HRESULT 패턴
if (FAILED(D3D11CreateDevice(...)))
{
    LogManager::Instance().Fatal("Create Device Error..!");
    return false;
}

HRESULT는 long 타입. 0 이상 = 성공, 음수 = 실패. DX11 코드에서 수십 번 반복되는 패턴이니까 익숙해져야 해.

Init — 초기화 6단계

Device::Init(hWnd, 1280, 720, true)
1
D3D11CreateDevice() — Device + Context 생성
2
CheckMultisampleQualityLevels() — 멀티샘플링 지원 확인
3
SwapChain 생성 — DXGI 체인으로 팩토리 → 스왑체인
4
GetBuffer() + CreateRenderTargetView() — 백버퍼 → RTV
5
CreateTexture2D() + CreateDepthStencilView() — 깊이/스텐실
6
RSSetViewports() — 뷰포트 설정

1. D3D11CreateDevice

C++ Device + Context 생성 Core/Device.cpp
uint32 flag = D3D11_CREATE_DEVICE_BGRA_SUPPORT;

#ifdef _DEBUG
    flag |= D3D11_CREATE_DEVICE_DEBUG;
#endif

D3D_FEATURE_LEVEL fLevel = D3D_FEATURE_LEVEL_11_0;
D3D_FEATURE_LEVEL fLevelResult;

if (FAILED(D3D11CreateDevice(
    nullptr,                    // 어댑터 (nullptr = 기본 GPU)
    D3D_DRIVER_TYPE_HARDWARE,   // 하드웨어 가속
    0,                          // 소프트웨어 래스터라이저 (안 씀)
    flag,                       // 플래그 (BGRA + DEBUG)
    &fLevel,                    // 요청 기능 레벨
    1,                          // 기능 레벨 배열 크기
    D3D11_SDK_VERSION,          // SDK 버전
    _device.GetAddressOf(),     // [out] Device
    &fLevelResult,              // [out] 실제 기능 레벨
    _context.GetAddressOf()     // [out] Context
)))

2. 멀티샘플링 (Anti-Aliasing)

C++ 4x MSAA 지원 확인 Core/Device.cpp
uint32 sampleCount = 4;
uint32 check = 0;
_device->CheckMultisampleQualityLevels(DXGI_FORMAT_R8G8B8A8_UNORM, 4, &check);

if (check < 1)
    sampleCount = 1;

멀티샘플링 = 계단 현상(jagging) 줄이기. 4x MSAA 요청하되, GPU가 지원 안 하면 1x로 폴백.

3. SwapChain 생성

C++ SwapChain 설정 Core/Device.cpp
DXGI_SWAP_CHAIN_DESC swapDesc = {};

swapDesc.BufferDesc.Width = widht;                          // 백버퍼 가로
swapDesc.BufferDesc.Height = height;                        // 백버퍼 세로
swapDesc.BufferDesc.Format = DXGI_FORMAT_R8G8B8A8_UNORM;    // RGBA 각 8비트
swapDesc.BufferDesc.RefreshRate.Numerator = 1;
swapDesc.BufferDesc.RefreshRate.Denominator = 60;            // 60Hz
swapDesc.BufferDesc.Scaling = DXGI_MODE_SCALING_UNSPECIFIED;
swapDesc.BufferDesc.ScanlineOrdering = DXGI_MODE_SCANLINE_ORDER_UNSPECIFIED;

swapDesc.BufferCount = 1;                                    // 백버퍼 1개
swapDesc.BufferUsage = DXGI_USAGE_RENDER_TARGET_OUTPUT;      // 렌더 타겟으로 사용
swapDesc.OutputWindow = _hWnd;                               // 출력할 윈도우
swapDesc.SampleDesc.Quality = 0;
swapDesc.SampleDesc.Count = sampleCount;                     // 멀티샘플링
swapDesc.Windowed = _windowMode;                             // 창모드 / 전체화면
swapDesc.SwapEffect = DXGI_SWAP_EFFECT_DISCARD;              // 이전 내용 무시
C++ Device → DXGI Factory → SwapChain Core/Device.cpp
// Device → IDXGIDevice → IDXGIAdapter → IDXGIFactory
ComPtr<IDXGIDevice> dxgiDevice = nullptr;
_device->QueryInterface(__uuidof(IDXGIDevice), (void**)dxgiDevice.GetAddressOf());

ComPtr<IDXGIAdapter> adapter = nullptr;
dxgiDevice->GetParent(__uuidof(IDXGIAdapter), (void**)adapter.GetAddressOf());

ComPtr<IDXGIFactory> factory = nullptr;
adapter->GetParent(__uuidof(IDXGIFactory), (void**)factory.GetAddressOf());

if (FAILED(factory->CreateSwapChain(_device.Get(), &swapDesc, _swapChain.GetAddressOf())))
{
    LogManager::Instance().Fatal("CreateSwapChain Error..!");
    return false;
}
왜 D3D11CreateDevice 하나로 안 만드나?
DXGI = 하드웨어 추상화 (어댑터, 모니터, SwapChain — "화면 관련")
DX11 = 렌더링 (Device, Context — "그리기 관련")

SwapChain은 "화면에 출력"하는 거라 DXGI 영역이고, D3D11CreateDevice는 "그리기 도구"만 만듦. 서로 다른 레이어라 분리되어 있어.

4. RenderTargetView — 백버퍼를 캔버스로

C++ 백버퍼 → RTV Core/Device.cpp
ComPtr<ID3D11Texture2D> backBuffer = nullptr;
if (FAILED(_swapChain->GetBuffer(0, __uuidof(ID3D11Texture2D), (void**)backBuffer.GetAddressOf())))
{
    LogManager::Instance().Fatal("_swapChain->GetBuffer Error!");
    return false;
}

if (FAILED(_device->CreateRenderTargetView(backBuffer.Get(), nullptr, _targetView.GetAddressOf())))
{
    LogManager::Instance().Fatal("CreateRenderTargetView Error!");
    return false;
}

5. DepthStencil — 깊이 + 스텐실 버퍼

C++ 깊이/스텐실 버퍼 생성 Core/Device.cpp
D3D11_TEXTURE2D_DESC depthDesc = {};
depthDesc.Width = widht;
depthDesc.Height = height;
depthDesc.ArraySize = 1;
depthDesc.Format = DXGI_FORMAT_D24_UNORM_S8_UINT;  // D24: 깊이 24비트, S8: 스텐실 8비트
depthDesc.BindFlags = D3D11_BIND_DEPTH_STENCIL;
depthDesc.Usage = D3D11_USAGE_DEFAULT;
depthDesc.SampleDesc.Count = sampleCount;
depthDesc.SampleDesc.Quality = 0;
depthDesc.MipLevels = 1;

ComPtr<ID3D11Texture2D> depthBuffer = nullptr;
if (FAILED(_device->CreateTexture2D(&depthDesc, nullptr, depthBuffer.GetAddressOf())))
{
    LogManager::Instance().Fatal("CreateTexture2D Error!");
    return false;
}

if (FAILED(_device->CreateDepthStencilView(depthBuffer.Get(), nullptr, _depthStencil.GetAddressOf())))
{
    LogManager::Instance().Fatal("CreateDepthStencilView Error!");
    return false;
}
깊이 vs 스텐실
깊이 (D24) — 각 픽셀의 카메라 거리 (0~1). 가까운 물체가 먼 물체를 가리는 데 사용.
스텐실 (S8) — 각 픽셀에 정수 값(0~255) 저장. 특정 영역만 그리기 같은 특수 효과.

6. Viewport

C++ 뷰포트 설정 Core/Device.cpp
D3D11_VIEWPORT vp = {};
vp.Width = static_cast<float>(widht);
vp.Height = static_cast<float>(height);
vp.MaxDepth = 1;

_context->RSSetViewports(1, &vp);

렌더링 결과를 화면의 어느 영역에 표시할지. 보통 전체 화면(1280x720). 화면 분할에서는 일부만 사용.

SetTarget + Render — 매 프레임

C++ 매 프레임 호출 Core/Device.cpp
void Device::SetTarget()
{
    _context->OMSetRenderTargets(1, _targetView.GetAddressOf(), _depthStencil.Get());
}

void Device::Render()
{
    _swapChain->Present(0, 0);
}
  • OMSetRenderTargets — "이 RTV와 DepthStencil에 그려라" (OM = Output Merger)
  • Present(0, 0) — 백 ↔ 프론트 버퍼 교체. 첫 번째 인자 0 = VSync 안 기다림

InitManager 변경

C++ Device 초기화 추가 Core/GameEngine.cpp
if (false == Device::Instance().Init(_hWnd, 1280, 720, true))
{
    LogManager::Instance().Fatal("Device Init Error!");
    return false;
}

Device는 InitManager의 마지막에 초기화 — 다른 매니저가 다 준비된 후에 GPU 초기화.
