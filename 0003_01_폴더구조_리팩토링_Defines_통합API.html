Perfect Forwarding

C++ New 함수 시그니처 Core/Defines.h
template<typename T, typename ...Args>
Ptr<T> New(Args&&... args)
{
    return new T(std::forward<Args>(args)...);
}

이 4줄에 고급 문법 3개가 들어있어:

  • typename ...Args — 가변 인자 템플릿 (인자 개수 제한 없음)
  • Args&&... args — 전달 참조 (lvalue/rvalue 모두 받음)
  • std::forward<Args>(args)... — 인자의 원래 종류를 보존해서 전달
forward 없이 (성능 손해)

return new T(args...);
rvalue가 lvalue로 변질 → 복사 생성자 호출

forward 있으면 (원본 유지)

return new T(std::forward<Args>(args)...);
rvalue 그대로 유지 → 이동 생성자 호출

자세한 설명은?
lvalue/rvalue, 참조 축약, forward의 동작 원리는 0003_02 — Perfect Forwarding 해부에서 깊이 다룸.

ObjectPool — type_index로 키 변경

이전 — enum 키
map<eObjectType, queue<Object*>>

새 타입 추가 → enum 수정 필요
Rent(eObjectType) → 타입을 2번 명시
현재 — type_index 키
map<std::type_index, queue<Object*>>

새 타입 추가 → 아무것도 안 해도 됨
Rent<T>() → 타입 1번만 명시
C++ Rent — type_index로 풀 찾기 Core/ObjectPool.h
template<typename T>
T* Rent()
{
    auto pool = _Pools.find(std::type_index(typeid(T)));
    if (pool == _Pools.end())
        return nullptr;

    T* retVal = nullptr;
    if (pool->second.empty())
    {
        retVal = new T();
        return retVal;
    }
    else
    {
        retVal = dynamic_cast<T*>(pool->second.front());
        if (nullptr == retVal)
            return nullptr;

        pool->second.pop();
    }

    return retVal;
}
C++ PreAlloc — 사전 할당 템플릿 Core/ObjectPool.h
template<typename T>
void PreAlloc(uint32 count)
{
    for (uint32 i = 0; i < count; ++i)
    {
        T* obj = new T();
        _Pools[std::type_index(typeid(T))].push(obj);
    }
}

// 사용:
void ObjectPool::Init()
{
    PreAlloc<TestObject>(10);
    // PreAlloc<Monster>(20);  ← 예시: 소스에는 없음. 나중에 이렇게 추가 가능
}
type_index 자세히 알고 싶으면?
RTTI, typeid, type_index의 동작 원리와 성능은 0003_03 — RTTI + type_index에서 깊이 다룸.
