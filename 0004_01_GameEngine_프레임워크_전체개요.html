GameEngine 클래스 구조

C++ GameEngine 선언 Core/GameEngine.h
class GameEngine : public Singleton<GameEngine>
{
    friend Singleton<GameEngine>;

protected:
    GameEngine() = default;
    ~GameEngine() = default;
    GameEngine(const GameEngine&) = delete;
    GameEngine(GameEngine&&) = delete;
    GameEngine& operator=(const GameEngine&) = delete;
    GameEngine& operator=(GameEngine&&) = delete;

private:
    HINSTANCE _hInst = nullptr;  // 운영체제가 발급해주는 ID
    HWND _hWnd = nullptr;        // 창마다 생기는 핸들
    HDC  _hdc = nullptr;         // 그리기 위한 핸들
    TCHAR _className[256] = {};
    TCHAR _titleName[256] = {};
    float _clearColor[4] = { 255, 255, 255, 255 };

    static bool _bIsRun;

public:
    virtual void Destroy() override;
    bool Init(HINSTANCE inst, const wchar_t* name);
    int Run();

    HDC GetHDC() const { return _hdc; }
    HINSTANCE GetHINSTANCE() const { return _hInst; }
    HWND GetHWND() const { return _hWnd; }

private:
    bool Create();
    void RegisterWindowClass();
    void Logic();
    void Tick(float deltaTime);
    bool InitManager();

    static LRESULT CALLBACK WndProc(HWND hWnd, UINT message, WPARAM wParam, LPARAM lParam);
};

InitManager — 매니저 초기화 순서

C++ 초기화 순서 Core/GameEngine.cpp
bool GameEngine::InitManager()
{
    DirectoryManager::Instance().Init();   // 1) 경로 먼저
    LogManager::Instance().Init();          // 2) 로그 (경로 필요)

    if (false == TimeManager::Instance().Init())  // 3) 시간
    {
        LogManager::Instance().Fatal("TimeManager Init Error!");
        return false;
    }

    return true;
}
매니저 의존 순서 — 왜 이 순서인가?
1
DirectoryManager — LogManager가 "Logs/" 경로를 물어봐야 함
2
LogManager — TimeManager 실패 시 에러 로그를 남겨야 함
3
TimeManager — 실패하면 false → _bIsRun = false → 엔진 정지

Singleton 변경 — Init() 순수 가상 제거

이전
virtual void Init() = 0;
virtual void Destroy() = 0;
현재
virtual void Init() = 0;
virtual void Destroy() = 0;

Init() 제거 이유 — 각 매니저의 시그니처가 전부 달라:

매니저Init 시그니처
GameEnginebool Init(HINSTANCE, const wchar_t*)
TimeManagerbool Init()
DirectoryManagervoid Init()
LogManagervoid Init()
MemoryPoolbool Init()
ObjectPoolbool Init()

순수 가상으로 강제할 수 없으니 제거. Destroy()는 전부 void+인자 없음이라 유지.

WndProc가 왜 static인가?

C++ static WndProc Core/GameEngine.h
static LRESULT CALLBACK WndProc(HWND hWnd, UINT message, WPARAM wParam, LPARAM lParam);
일반 멤버 함수라면?
LRESULT (GameEngine::*)(HWND, UINT, WPARAM, LPARAM)

↑ 숨겨진 this가 있어서
OS가 기대하는 타입과 불일치!
static 멤버 함수
LRESULT (*)(HWND, UINT, WPARAM, LPARAM)

this 없음 → 일반 함수 포인터와 호환!
wcex.lpfnWndProc = WndProc; OK
_bIsRun이 static인 이유
static WndProc에서 멤버 변수에 접근할 수 없으니, 엔진 종료 플래그 _bIsRun도 static으로 선언. WM_DESTROY → _bIsRun = false → 게임 루프 탈출.

AdjustWindowRect — 1280x720 맞추기

C++ Create() Core/GameEngine.cpp
bool GameEngine::Create()
{
    _hWnd = CreateWindowW(_className, _titleName, WS_OVERLAPPEDWINDOW,
        CW_USEDEFAULT, 0, CW_USEDEFAULT, 0, nullptr, nullptr, _hInst, nullptr);

    if (!_hWnd)
    {
        return false;
    }

    RECT windowRect = { 0, 0, 1280, 720 };

    // 윈도우 클라이언트 영역을 1280x720에 맞춘다
    AdjustWindowRect(&windowRect, WS_OVERLAPPEDWINDOW, false);
    SetWindowPos(_hWnd, HWND_TOPMOST, 0, 0,
        windowRect.right - windowRect.left,
        windowRect.bottom - windowRect.top,
        SWP_NOMOVE | SWP_NOZORDER);

    ShowWindow(_hWnd, SW_SHOW);
    UpdateWindow(_hWnd);
    return true;
}
🖼️ ANALOGY — 액자 비유
CreateWindowW(1280, 720) = 액자 틀 포함 1280x720. 실제 그림 공간은 더 작아.
AdjustWindowRect() = "그림 공간이 정확히 1280x720이 되려면 액자가 얼마나 커야 하는지" 계산.
