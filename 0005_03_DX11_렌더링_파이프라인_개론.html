왜 이 순서인가?

🏭 ANALOGY — 공장 라인
1. 재료 준비 (IA) — 나무 블록(정점)을 라인에 올림
2. 모양 잡기 (VS) — 블록의 위치/크기를 조정
3. 자르기 (RS) — 3D 모형을 2D 사진으로 찍음
4. 색칠하기 (PS) — 사진의 각 픽셀에 색을 입힘
5. 완성품 검수 (OM) — 앞에 있는 건 보이고, 뒤에 있는 건 가리고

이 순서가 아니면 안 돼 — 위치를 모르면 어디를 자를지 모르고, 자르지 않으면 어떤 픽셀에 색칠할지 모르니까.

파이프라인 각 단계

FIXED Input Assembler IA
"삼각형의 재료를 준비해"
입력: Vertex Buffer (정점), Index Buffer (인덱스), Input Layout
하는 일: GPU가 처리할 수 있는 형태로 정점 데이터를 조립

인덱스 버퍼로 정점을 공유할 수 있어 — 사각형은 정점 4개 + 인덱스 6개 (중복 없이).
PROGRAMMABLE Vertex Shader VS
"각 꼭짓점을 변환해"
입력: 각 정점 (모델 공간)  →  출력: 변환된 정점 (클립 공간)
HLSL로 직접 코드 작성. GPU에서 모든 정점에 대해 병렬 실행.

화면 기준
OPTIONAL Hull / Tessellator / Domain / Geometry HS TS DS GS
테셀레이션 — 폴리곤을 더 잘게 쪼개 부드럽게. 지오메트리 셰이더 — 프리미티브 단위로 정점 추가/삭제.
현재 프로젝트에서는 사용 안 함.
FIXED Rasterizer RS
"삼각형을 픽셀로 변환해"
입력: 클립 공간의 삼각형  →  출력: 화면 위의 픽셀 후보 (프래그먼트)
프로그래밍 불가, GPU 자동 처리. 여기서 Viewport (RSSetViewports)가 사용됨.
PROGRAMMABLE Pixel Shader PS
"각 픽셀의 색깔을 결정해"
입력: 각 픽셀 후보  →  출력: 최종 색깔 (RGBA)
텍스처 샘플링, 조명, 그림자, 반사. HLSL로 작성, GPU에서 모든 픽셀에 병렬 실행.
FIXED Output Merger OM
"최종 결과를 화면에 찍어"
깊이 테스트 — 앞에 있는 픽셀만 남김
스텐실 테스트 — 특정 영역 마스킹
블렌딩 — 반투명 오브젝트 색 섞기

여기서 RenderTargetView와 DepthStencilView가 사용됨.
