TimeManager +
deltaTime + chrono

프레임 간격은 일정하지 않다. 그 간격을 측정하는 게 TimeManager. deltaTime이 왜 필요하고, chrono가 어떻게 동작하는지.

deltaTime이 뭔가?

deltaTime = 이전 프레임과 현재 프레임 사이의 시간 차이 (초 단위)

프레임별 deltaTime 변화
0.016s
60fps
→
0.017s
59fps
→
0.033s
30fps
→
0.016s
60fps

프레임마다 걸리는 시간이 다르다. 30fps 프레임은 60fps보다 2배 오래 걸림.

왜 필요한가? — 프레임 독립 움직임

deltaTime 없이
position += 5; (매 프레임 5px)

60fps → 초당 300px
30fps → 초당 150px

같은 게임인데 속도가 달라!
deltaTime 곱하면
position += 300.0f * deltaTime;

60fps → 300 × 0.016 = 4.8px/프레임
30fps → 300 × 0.033 = 9.9px/프레임

둘 다 초당 ~300px. 동일!
🏃 ANALOGY — 달리기 비유
"매 발걸음 5미터"가 아니라 "초속 300미터"로 정의하면,
발걸음이 빠르든 느리든 같은 시간에 같은 거리를 이동해.

deltaTime은 "이번 발걸음이 몇 초짜리인지" 알려주는 스톱워치야.

std::chrono 타입 계층

chrono의 3가지 핵심 타입
1
clock (시계) — 시간을 측정하는 원천. high_resolution_clock = 가장 정밀한 시계
2
time_point (시각) — "언제". 특정 시점의 타임스탬프. clock::now()로 얻음
3
duration (기간) — "얼마나". 두 time_point의 차이. duration<float> = 초 단위 실수
clock 종류기준용도
system_clock1970.01.01 (epoch)시스템 시간 (날짜/시간 표시)
steady_clock부팅 시점경과 시간 측정 (되감기 불가)
high_resolution_clock보통 steady_clock과 동일가장 정밀한 시계 (게임용)
왜 system_clock이 아니라 high_resolution_clock?
게임에서는 "지금 몇 시"가 아니라 "이전 프레임에서 얼마나 지났는지"가 중요. system_clock은 사용자가 시간을 바꾸면 되감기 될 수 있어서 게임 타이밍에 부적합.

TimeManager 전체 코드

C++ 헤더 Core/TimeManager.h
#pragma once
#include "Common/Singleton.h"
#include <chrono>

class TimeManager : public Singleton<TimeManager>
{
    friend Singleton<TimeManager>;

protected:
    TimeManager() = default;
    ~TimeManager() = default;
    TimeManager(const TimeManager&) = delete;
    TimeManager(TimeManager&&) = delete;
    TimeManager& operator=(const TimeManager&) = delete;
    TimeManager& operator=(TimeManager&&) = delete;

private:
    std::chrono::high_resolution_clock::time_point _prevTime;
    float _deltaTime = 0.f;

public:
    bool Init();
    virtual void Destroy() override;
    float Tick();
    float GetDeltaTime() const { return _deltaTime; }
};
C++ Init — 시작 시각 기록 Core/TimeManager.cpp
bool TimeManager::Init()
{
    _prevTime = std::chrono::high_resolution_clock::now();
    return true;
}

Tick — deltaTime 계산

C++ Tick() Core/TimeManager.cpp
float TimeManager::Tick()
{
    auto curTime = std::chrono::high_resolution_clock::now();
    std::chrono::duration<float> frameTime = curTime - _prevTime;
    _deltaTime = frameTime.count();
    _prevTime = curTime;

    return _deltaTime;
}
Tick() 한 줄씩 해부
1
auto curTime = ...::now(); — 현재 시각 찍기 (time_point)
2
duration<float> frameTime = curTime - _prevTime; — 두 시각의 차이 = duration (자동 초 단위 float 변환)
3
_deltaTime = frameTime.count(); — duration에서 float 값 꺼내기
4
_prevTime = curTime; — 현재 시각을 "이전 시각"으로 갱신 (다음 Tick 준비)
시간 축에서 보는 Tick()
_prevTime
frameTime (duration<float>)
curTime
↓ _prevTime = curTime (갱신)
_prevTime
다음 프레임의 frameTime
next now()
duration<float>의 의미
duration<float>는 "초 단위 실수"로 자동 변환해줘. duration<double, std::milli>면 밀리초 단위가 됨. 템플릿 인자로 단위를 제어하는 거야.

GameEngine에서의 사용

C++ Logic/Tick Core/GameEngine.cpp
void GameEngine::Logic()
{
    float deltaTime = TimeManager::Instance().Tick();
    Tick(deltaTime);
}

void GameEngine::Tick(float deltaTime)
{
    LogManager::Instance().Tick();

}

현재 Tick()에는 LogManager만 있지만, 앞으로 입력 처리, 오브젝트 업데이트, 렌더링 등이 여기 추가될 예정.

GameEngine::Run() └─ while (_bIsRun) └─ Logic() ├─ deltaTime = TimeManager::Instance().Tick() └─ Tick(deltaTime) └─ LogManager::Instance().Tick()
