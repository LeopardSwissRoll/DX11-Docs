예시: 약화 상태에서 Strike 사용 vs 취약 적

플레이어에게 약화(2턴)가 걸려있다. 적에게 취약(1턴)이 걸려있다. 플레이어가 Strike (6데미지, 1에너지)를 사용한다.

# 주체 동작 데이터
1 플레이어 Strike 선택, 대상 Enemy[0]
2 카드 시스템 에너지 확인: 3 ≥ 1 (비용). 에너지 차감. energy: 3 → 2
3 EventBus BeforeCardPlayed 발행 card=Strike, target=Enemy[0]
4 카드 시스템 EffectDef 읽기 → Action{Damage, 6} 생성
5 ActionQueue pushBack(Action{Damage, src=Card(Strike), target=E0, value=6}) 큐: [Damage(6)]
6 EventBus CardPlayed 발행 트리거 확인 (쿠나이 등) — 없음
7 카드 시스템 Strike를 버린 더미로 이동 손패 → 버린 더미
▼ 액션 소비 시작 ▼
8 ActionQueue pop() → Damage 액션 꺼냄 type=Damage, value=6, src=Card
9 전투 해결기 DamageRequest{amount=6, type=Attack} 생성
10 EventBus BeforeDamage 발행 → 인터셉터가 req 수정
11 약화 [p=200] 공격자가 플레이어 + 유형이 Attack → amount × 0.75 amount: 6 → 4 (내림)
12 취약 [p=400] 대상에 취약 + 유형이 Attack → amount × 1.5 amount: 4 → 6 (내림)
13 전투 해결기 피해 적용: 방어도(0) 흡수 0, HP에 6 적용 적 HP: 46 → 40
14 EventBus AfterDamage 발행 반응형 트리거 없음 — 큐 비어있음
15 ActionQueue 큐 비어있음. 플레이어 입력으로 복귀.

셔플: RNG 트랙 사용

RNG 격리: 셔플은 전용 shuffleRng 트랙을 사용 (전투별 시드). 카드 보상은 cardRewardRng. 랜덤 타겟팅은 miscRng. 이렇게 하면 셔플 순서가 같은 시드에서 결정적이며, 다른 랜덤 이벤트의 영향을 받지 않는다.

현재 DX_Mine의 Random 클래스는 단일 mt19937. STS용으로는 이름별 트랙 여러 개를 감싸면 된다: RngTracks { mt19937 shuffle, cardReward, misc, enemy; }

enum class Op { Damage, GainBlock, ApplyStatus, Draw, Discard, Exhaust, AddEnergy, LoseHP, Heal }; struct ValueExpr { ValueKind kind; // Const, UpgradedPair, XCost, // RandomRange, PerStatus int a = 0, b = 0; StatusId refStatus{}; }; struct EffectDef { Timing when; // OnPlay (기본값), OnDraw... TargetSelector target; // Self, ChosenEnemy, All... Op op; // 연산 종류 ValueExpr value; // 값 표현식 int repeat = 1; // 반복 횟수 TagSet tags; // 태그 집합 };

struct Action { enum class Type { Damage, GainBlock, ApplyStatus, RemoveStatus, Draw, Discard, Exhaust, AddCardToHand, AddEnergy, LoseHP, Heal, ShuffleDiscardIntoDraw, RollEnemyIntent, WaitForInput, EndTurn, }; Type type; // 액션 종류 EffectSource source; // 발신자 EntityId targetId; // 대상 int value = 0; // 값 StatusId statusId{}; // 상태이상 ID (해당 시) CardInstance* card = nullptr; // 카드 (해당 시) };

struct CardDef { CardId id; string name; // 카드 이름 CardType type; // 공격/스킬/파워/상태/저주 int baseCost; // 기본 에너지 비용 TargetRule targetRule; // 대상 규칙 vector<EffectDef> effects; // 효과 목록 CardFlags flags; // 소멸/천상/선천/유지 CardId upgradedVersion; // 업그레이드 버전 ID }; struct CardInstance { const CardDef* def; // 정의 참조 int costDelta = 0; // 전투 중 비용 변동 bool upgraded = false; // 업그레이드 여부 bool retain = false; // 이번 턴 유지 여부 InstanceId iid; // 인스턴스 고유 ID };

enum class CardType { Attack, Skill, Power, Status, Curse }; // 공격, 스킬, 파워, 상태, 저주 enum class DamageType { Attack, HP_Loss, Thorns }; // 공격, HP 손실, 가시 enum class TargetRule { Self, SingleEnemy, AllEnemies, Random, None }; // 자신, 단일적, 전체적, 무작위, 없음 enum class StatusCategory { Buff, Debuff, Special }; // 버프, 디버프, 특수 enum class Timing { OnPlay, OnDraw, OnDiscard, OnExhaust, OnTurnStart, OnTurnEnd }; // 사용 시, 드로우 시, 버릴 때, 소멸 시, // 턴 시작, 턴 종료 enum class ValueKind { Const, UpgradedPair, XCost, RandomRange, PerStatus, PercentOf }; // 상수, 업그레이드쌍, X비용, // 랜덤범위, 상태당, 퍼센트
